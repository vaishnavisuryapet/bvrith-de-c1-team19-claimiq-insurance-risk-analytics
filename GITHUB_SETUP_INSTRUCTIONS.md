# GitHub Setup Instructions for Student Teams

## Option 1 — Preferred: Use as a GitHub Template

Use this option whenever the official ZENAIZ template repository is available.

1. Open the ZENAIZ starter repo.
2. Click **Use this template**.
3. Create a new repository using the naming format:

```text
bvrith-de-c1-teamXX-[short-project-name]
```

Example:

```text
bvrith-de-c1-team04-ipl-matchday-360
```

4. Add all three students as collaborators.
5. Update `README.md`.
6. Commit Week 1 files.

---

## Option 2 — Backup Only: Download ZIP and Create New Repo

Use this option only if the GitHub template button is not available or the mentor specifically asks the team to use the ZIP method.

1. Download the starter repo ZIP.
2. Extract it locally.
3. Open the extracted folder and make sure the repo files are visible directly at the root.
4. Create a new empty GitHub repo.
5. Upload the starter files and folders directly into the repo root.
6. Update `README.md`, `docs/problem_charter.md`, and `weekly_logs/week01_log.md`.

Do not upload a wrapper folder such as `de_repo/` or `zenaiz-de-student-project-template-v3-final/` into GitHub.

---

## Naming Rules

Use simple lowercase names:

```text
bvrith-de-c1-team01-urban-mobility
bvrith-de-c1-team02-ipl-matchday
bvrith-de-c1-team03-weather-command-center
```
