# START HERE — ZENAIZ Data Engineering Starter Repo

This repository already contains the student templates.

There is no separate template download required.

## What does “template” mean here?

A template is a placeholder file that students fill week by week.

For example:

```text
README.md
docs/problem_charter.md
docs/data_dictionary.md
weekly_logs/week01_log.md
final_submission/final_report.md
```

These are the templates.

They are placed in the same folders where the final work should remain.

---

## What should students do in Week 1?

Update only these files:

```text
README.md
docs/problem_charter.md
weekly_logs/week01_log.md
```

Do not complete all files in Week 1.

---

## What should students do every week?

1. Open the weekly sprint sheet.
2. Check which files to update.
3. Update only those files.
4. Save screenshots if needed.
5. Commit and push to GitHub.
6. Be ready for selected evidence review.

---

## Mandatory AI Transparency Note

AI can be used as a learning and engineering support tool, but the team must remain responsible for the work.

Every weekly log must clearly mention:

- where AI was used,
- what the team changed or corrected,
- what was manually verified,
- what each student can explain without AI.

Do not submit AI-generated content, code, explanations, dashboards, or documentation that the team cannot explain during review.

---

## Important

Students should not create a separate `templates/` folder in their final project.

The working placeholder files inside this repo are the templates.
